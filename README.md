# Relational-Database-Architecture-for-Business-Operations-SQL-Server-SSMS-

   This project consists of a comprehensive SQL Server database named Firma_MEA, designed to streamline corporate operations by managing human resources, product inventory, and sales transactions. It implements a robust relational schema with integrated business logic through custom functions and views.

Technical Features
Database Engine: SQL Server (T-SQL).

Relational Design: 7 interconnected tables with enforced referential integrity.

Advanced Logic: Implementation of Table-valued functions, Scalar functions, and complex Joins.

Data Abstraction: Custom views for simplified reporting.


The system is built around the following core modules:

1.Human Resources (HR)

DEPARTAMENTE: Stores organizational units (e.g., Management, Production, Sales).

FUNCTII: Defines job roles and base salaries with CHECK constraints to ensure positive values.

ANGAJATI: Manages employee records, linking them to departments and specific job functions.

2.Product & Inventory Management

CATEGORII_PROD: Organizes products into groups like Cables, Adapters, and Power Supplies.

PRODUSE: Detailed product list linked to their respective categories.

3.Sales & CRM
CLIENTI: Tracks clients, their company types (SRL, SA, etc.), and geographical locations.

VANZARI: Records every transaction, linking products, clients, and the specific salesperson (employee).


The database includes several programmed objects to automate data analysis:

Employee Insights: The f_Angajati_Functie_Contine function allows filtering staff based on specific keywords in their job titles.

Financial Reporting: The f_MinMax function provides instant minimum and maximum salary data for any given department.

Sales Performance: f_Suma_Vanzator: Calculates the total revenue generated by a specific salesperson.

f_Depaseste_Prag: A logic gate (Boolean) that checks if a salesperson has exceeded a specific performance threshold.

Data Abstraction: The vAngajati view provides a clean, joined interface of employee names, departments, and salaries, hiding the underlying complexity of multiple joins.


Conclusion:
This database serves as a complete backend solution for managing a company's core operations. By centralizing HR, inventory, and sales, it transforms raw data into a structured environment ready for professional reporting and decision-making.
